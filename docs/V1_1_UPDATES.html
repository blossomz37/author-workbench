Last update: 2026-05-24. This page summarizes the private working repo state in public-safe terms.

What changed in V1.1 so far

done Hybrid stream reattachment Project-aware runs now emit through a live in-memory buffer. A returning client can reattach by run ID and event index without starting a second provider call.
done Transient retry policy Provider text streams retry transient failures twice with exponential backoff. Fatal errors, including missing keys, still fail immediately.
done Smoke gate added npm run smoke:resume verifies mid-stream client drop, reattach, run-folder continuity, and mock transient retry evidence.
done Multi-phase runner Actions can execute ordered phases, pause for review, resume with approved phase output, and save phase-level artifacts.
done Overhaul variants Manuscript and dossier overhaul actions now cover single, multiple, and all-file input modes plus selected pass execution.
done Workflow contracts Visible actions now declare editable templates, review behavior, checkpoint behavior, run-to-phase support, and supported review decisions.
done Aurora Prism M2 route pass The live app now applies the approved design foundation across project opening, dashboard, runner, live run, review/results, and settings screens.
done Editorial skill expansion Grumpy Gus Chapter Critique and Prose Gate Validation now run as app-ready skill actions with prompt adapters and declared run artifacts.
done Run-history browsing The dashboard can browse, filter, search, and page through saved runs across multiple action types.

Decision log

V1.1 scope confirmed Reliability, design implementation, and skill expansion are in. Anthropic adapter is explicitly out for this version.
Reattachment architecture selected Use a hybrid model: live in-memory buffer for active streams, disk recovery from saved run evidence when the process or buffer is gone.
Retry policy selected Retry timeout/network reset/429/5xx-style failures twice. Do not retry bad keys, auth, malformed requests, validation errors, or refusal-style fatal errors.
Manuscript Overhaul surfaced as app actions The app now models manuscript modular, manuscript full, dossier modular, and dossier full as separate runnable actions.
Visible actions now require workflow contracts Each overhaul action maps to its own templates, phases, pause behavior, checkpoint behavior, and UI controls.
Aurora Prism M2 route pass selected and landed The next V1.1 checkpoint after workflow contracts was the route-level design pass, not M3 skill expansion.
FR-018 skill choices resolved Grumpy Gus Chapter Critique and Prose Gate Validation became the two V1.1 editorial skill actions.
