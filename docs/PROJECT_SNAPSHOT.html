One-Line Status

The core V1 proof is working: load files, assemble prompts, stream output, pause for review, resume to audit, and save the evidence locally.

What Is Proven

  • OKBrowser streaming works with Server-Sent Events.
  • OKLocal project files can be read into prompt context.
  • OKHandlebars templates assemble draft, audit, and skill-backed prompts.
  • OKRun folders preserve prompts, streamed events, drafts, approvals, audits, and outputs.
  • OKMock mode can test the workflow without an API key.

Fixtures

  • projects/dead-acre/ is the heavier real-world fixture.

What Changed Recently

Action layer added The app moved beyond one hardcoded draft/audit demo.
Runner types now exist draft-and-audit, single-step, and skill-template.
Strict schema validation Malformed action metadata fails early instead of breaking during preview or runtime.
Action metadata lives in project files For now, each project's actions are declared in project.yaml.

Design Bias

  • Local files as truth.
  • Visible prompt assembly.
  • Saved evidence over hidden runtime state.
  • Mock-first testing.
  • Explicit action wrappers for skills.

Still Open

  • Anthropic provider adapter.
  • Large dossier and context-window limits.
  • Multiple approval gates.
  • project.yaml vs actions.yaml.
  • Which real skills to wrap next.

How To Run

npm install
npm run dev:server
npm run dev:ui

Then open http://localhost:5173/. Mock mode works without an API key.
