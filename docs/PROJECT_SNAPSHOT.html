Quick Overview

Author Workbench is a local-first author workflow app for testing AI-assisted fiction workflows. It loads author project files, assembles prompts, streams mock or OpenAI output, pauses for human review, and saves each run as local evidence.

This public snapshot is intentionally small. It is a shareable check-in for friends comparing parallel approaches, not the full working source repo.

Current status: V1 scaffold and workflow proof. The core loop can load files, assemble prompts, stream output, pause for review, resume to audit, and save the evidence locally.

Agent/CLI Parity

The current V1 app has API-level parity with agent and terminal operation. Everything important the browser can do can also be driven by an agent through the Express API and local run folders.

This is not a polished standalone CLI yet. There is no author-workbench run command. The terminal path today is npm run dev:server plus direct API calls.

  • APIOpen or validate a project with GET /api/project?path=....
  • APIPreview assembled prompts with POST /api/prompt/preview.
  • SSEStart and watch workflow runs with GET /api/stream?... and a streaming client like curl -N.
  • APIApprove edited drafts with POST /api/run/approve, or cancel paused runs with POST /api/run/cancel.
  • FILESInspect completed work by reading the timestamped local runs/ folder.

What Changed Recently

The project started as a single draft-and-audit proof. It now has a small app-action layer so different author workflows can become visible, testable actions.

Action layer added The app moved beyond one hardcoded draft/audit demo.
Runner types now exist draft-and-audit, single-step, and skill-template.
Strict schema validation Malformed action metadata fails early instead of breaking during preview or runtime.
Action metadata lives in project files For now, each project's actions are declared in project.yaml.
Support Skills In This Workspace
  • specforge: Turns fuzzy app or workflow ideas into evidence-backed build specs.
  • skill-to-app-wizard: WIP method for converting a readable agent skill into a testable app action.
  • express-svelte-monorepo: Scaffold pattern for the current full-stack shape.
  • sse-streaming: Reusable SSE event and streaming patterns for Express and browser clients.
  • svelte5-rune-store: Shared Svelte 5 state pattern using rune-backed getter/setter functions.

Note: skill-to-app-wizard is still under .agents/wip-skills/; the others are under .agents/skills/.

Design Bias
  • Local files as truth.
  • Visible prompt assembly.
  • Saved evidence over hidden runtime state.
  • Author approval before downstream audit.
  • Mock-first testing.
  • Simple infrastructure before a database or full workflow engine.
  • Explicit action wrappers for skills.
Still Open
  • Anthropic provider adapter.
  • Large dossier and context-window limits.
  • Multiple approval gates.
  • project.yaml vs actions.yaml.
  • Which real skills to wrap next.
  • How polished the UI needs to become before this is useful beyond internal testing.
How To Run 
npm install
npm run dev:server
npm run dev:ui

Then open http://localhost:5173/. Mock mode works without an API key. OpenAI mode requires OPENAI_API_KEY in the server environment.
